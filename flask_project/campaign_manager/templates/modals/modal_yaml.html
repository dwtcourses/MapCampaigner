Notes

- YAML is white-space sensitive. Follow the indentation of the example exactly.
- Values that contain spaces should be wrapped in double quotes, e.g. "My Custom Feature Name".
- The top-level parent is the title of the custom feature type.
- The second-level should have two keys: feature and tags.
- You can add more than one feature type at a time in the YAML editor.

Example

"All Restaurants":
  feature: amenity=restaurant
  tags:
    - name
    - cuisine
    - opening_hours
  element_type: Point
"Local Shops":
  feature: shop
  tags:
    - name
    - opening_hours
    - shop:
      - art
      - bakery
      - bicycle
  element_type: Polygon

 

- feature: The OpenStreetMap feature you want to track. For example: building, amenity, building=school or building=school,university.
- tags : This is a list of OpenStreetMap keys you want to track. Start every item with a dash (i.e. -). For example: name, operator, opening_hours. - element_type: Can be Point, Line, or Polygon.
